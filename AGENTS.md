# AGENTS.md — react-material-expressive

Contributor and agent guide. The library is a pure React UI kit implementing
**Material 3 Expressive**: presentational, framework-agnostic, with
precompiled CSS and runtime token theming. `CLAUDE.md` is just a pointer
(`@AGENTS.md`); this is the document to keep current.

## Start here

- [README.md](README.md) — consumer docs: install, quickstart, theming,
  icons, media/link injection, i18n, exports.
- [llms.txt](llms.txt) — LLM index of every `docs/components/*.md`.

## Architecture

```
src/
├── index.ts                  # root barrel ("use client") → re-exports components, elements, layers
├── styles.css                # precompiled-CSS source (tokens + utilities + classes)
├── theme.css                 # Tailwind v4 partial (@theme inline) — also published raw for extension
├── components/<folder>/      # one folder per component; sub-parts co-located
│   ├── _useDismissable.ts    # internal: overlays with exit animation
│   ├── _useOutsideClose.ts   # internal: close on outside click
│   └── _useFocusTrap.ts      # internal: modal focus-in/restore + Tab trap
├── elements/common/          # primitives (Amount, Checkbox, Switch, Img…)
│   └── _media.tsx            # internal: media-injection contract + native <img>
├── layers/                   # Container, Section
└── utils/                    # internal only (not exported): cn, getSrc, theme, navigation, _useControlled, _ripple
```

Single root export (one tsup entry): `.` (all components, elements and
layers), plus the precompiled `./styles.css` and the raw `./theme.css`
Tailwind v4 partial. The JS API is components + elements + layers only
(`utils/` stays internal); theming is CSS token overrides + the
`ToggleTheme`/`ToggleThemeMenu` components, and `theme.css` lets a consumer
extend the design with M3 utilities in their own Tailwind v4 (token values
still come from `styles.css`).

### Conventions

- **`_*` files** are internal — never exported from public barrels.
- **Child-only sub-components** live in the parent folder and are exposed as
  static properties (`Card.Header`, `Dialog.Footer`, `Menu.Item`, `*.Item`…).
- **React 19**: `ref` as a prop (no `forwardRef`); native props via
  `ComponentProps<"...">`.
- **Internal imports** by direct file path; **stories** import only public
  barrels.
- **Full Tailwind class literals** (never partial concatenation): the
  precompiled CSS is generated by scanning literals.
- **One utility per CSS property** (no shorthand + longhand on the same
  element): never pair `px-*` with `pl-*`/`pr-*` (or `py-*` with `pt-*`/`pb-*`,
  `p-*` with a side, etc.). A consumer's own Tailwind shares our `@layer
  utilities`; if it re-emits the shorthand (e.g. `.px-4`) after our stylesheet
  it overrides our longhand and the inset is lost (this is what put the text
  field's placeholder/value over the leading icon). Pick the value per side
  instead: `leftElement ? "pl-13" : "pl-4"`. Two same-side longhands
  (`pl-4` + `pl-12`) are fine — `cn`/tailwind-merge collapses them to one.
- **Icons** are always `ReactNode` props; zero icon libraries. Item icons sit
  in a fixed 24×24 box.
- **Images/links**: no Image/Link components. Media is injectable with
  priority `render > image > children > <img>` (`_media.tsx`); navigation uses
  native `<a>` + `active`/`currentPath`/`onClick`.
- **State** is ephemeral UI only. Controllable inputs use `useControlled`;
  Checkbox/Radio use the native input as the source of truth.
- **i18n via `labels`**: every component with visible text or an accessible
  name takes a `labels?: XLabels` object (optional strings) merged over an
  English-default `const X_LABELS`. The library ships English only; date/number
  formatting goes through a separate `locale` prop (`Intl`), never `labels`.
  Export the `XLabels` type from the barrel and document it.
- **Accessibility is owned by the component** (see below).
- **Expressive-only**: variants the spec marks "no longer recommended" are not
  shipped (see README + FOUNDATIONS).

## Accessibility

Interactive components own their semantics — don't push this onto consumers:

- **Focus**: modals (Dialog, sheets) move focus in on open, trap Tab/Shift+Tab
  and restore focus on close via the shared `_useFocusTrap`. Menus use roving
  focus.
- **ARIA**: correct roles and `aria-expanded`/`controls`/`selected`/`checked`/
  `modal`/`current` plus live regions where state changes must be announced.
  Popup ARIA goes on the real focusable trigger, not a wrapper.
- **Names**: every interactive element the library renders has an accessible
  name (its content, a `labels` value, or a forwarded `aria-label`).
- **Keyboard**: composite widgets (tabs, sliders, dial) are arrow-key
  operable; Escape dismisses overlays and tooltips.
- **Motion**: decorative infinite animations are gated under
  `prefers-reduced-motion`; essential motion (loading) is exempt.

Cover a11y-relevant behavior with tests (`*.test.tsx`, Vitest + Testing
Library + jest-axe; jsdom can't check contrast — that's verified visually).

## Documentation alongside changes

Any change must update: this file (if conventions/architecture change), the
touched `docs/components/*.md` (API, variants, gotchas, accepted gaps), and —
when adding/renaming/removing a component — `llms.txt` and the README catalog.
Never land a behavior change leaving docs that say otherwise.

## Commands

| Command                                 | What it does                                                      |
| --------------------------------------- | ----------------------------------------------------------------- |
| `npm run build`                         | clean → build:css (Tailwind CLI) → build:lib (tsup) → copy:themes |
| `npm run build:css`                     | compile `src/styles.css` → `dist/styles.css` (minified)           |
| `npm run build:lib`                     | tsup: ESM (.js) + CJS (.cjs) + dts for the root entry             |
| `npm run copy:themes`                   | copy raw `theme.css`, stamp `"use client"` on the entry           |
| `npm test` / `test:watch`               | Vitest (render + a11y assertions)                                 |
| `npm run typecheck` / `lint` / `format` | `tsc --noEmit` / ESLint (`--max-warnings=0`) / Prettier           |

## Build & publish

- The build compiles the CSS with the Tailwind CLI, then tsup emits
  ESM + CJS + dts for the single root entry; `scripts/copy-themes.mjs` copies
  `theme.css` raw and stamps `"use client"` at byte 0 of the entry (tsup's
  banner doesn't survive esbuild) — this is what makes the package RSC/SSR-safe.
- The published set (`files`) is `dist/`, `docs/`, `CHANGELOG.md`,
  `README.md`, `llms.txt`. `prepack` rebuilds `dist/`; `prepublishOnly` runs
  lint + typecheck + test. Published to public npm (`publishConfig.access:
public`).
- **CHANGELOG** (Keep a Changelog + SemVer): each release moves `[Unreleased]`
  to a versioned, dated section with a comparison link; every breaking change
  carries a **Migration** note.
